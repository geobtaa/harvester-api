Socrata Harvester

Purpose

This script scans the DCAT 1.1 APIs of Socrata portals and returns metadata for all suitable items. It produces two CSV files:

  • A Primary CSV containing all metadata except the link fields
  • A Distribution CSV with only the link fields (e.g., landing page, download, web services)

Prepare the list of active Socrata portals

Use the following shortcut query to filter for appropriate items:

Filter Socrata portals in GBL Admin

Export the results as a CSV file. You can upload this file directly using the interface below, and it will be placed in the data folder for harvesting.

Required Columns

The exported CSV from GBL Admin will contain many fields, but this script uses only the following:

  • ID: Unique code assigned to each portal. Transferred to the Is Part Of field for each dataset.
  • Title: The name of the Portal. Transferred to the Local Collection field for each dataset.
  • Publisher: The place or administration associated with the portal. Applied to the title of each dataset in brackets.
  • Spatial Coverage: A list of place names. Transferred to the Spatial Coverage for each dataset.
  • Bounding Box: Used as the fallback value if the harvested dataset does not have one.
  • Member Of: The ID of a larger collection-level record. Most Hubs are either part of the Government Open Geospatial Data Collection or the Research Institutes Geospatial Data Collection.
